Separation of Concerns

This page will explain what Separation of Concerns is. I will also provide an example Python program that shows what an implementation of Separation of Concerns looks like.

What is Separation of Concerns?

Separation of Concerns (SoC) s a design principle for separating a computer program into distinct sections. Each section addresses a separate concern, a set of information that affects the code of a computer program. A program that embodies SoC well is called a modular program. Modularity, and hence separation of concerns, is achieved by encapsulating information inside a section of code that has a well-defined interface.

For a solid explanation of separation of concerns, take a look at this StackOverflow post.

Separation of Concerns Example

Take a look at this GitHub repository of a simple calculator. This program implements separation of concerns.

The Calculator class is separated from each respective operations code. The history of calculations is separated into its own class. A general calculation is separated from more specific ones (addition, subtraction, multiplication, division). In doing so, when it comes time to change an aspect of the program, you need only go to the respective file for it. When changing the workings of the Addition class, for example, you don't need to change the workings of the history or the Calculator. In doing so, you are less likely to break unrelated code.
