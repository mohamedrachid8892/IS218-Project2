Object Oriented Programming Glossary

This page provides a glossary for key terms needed to understand OOP principles.

Glossary

  • Factory - An object used for creating other objects. Formally, a factory is a function or method that returns objects of varying class from some method call.
  • Instantiation - The process of taking a class definition and creating an object that you can use in a program.
  • Object - An abstract data type created by a developer. Objects are described by their state and behavior, stored in fields and exposed through methods, respectively. Objects are instances of classes.
  • Class - A template used for creating classes. Provides initial values for states and implementations of behavior through class methods.
  • Namespace - A region that provides a scope to the identifiers (names of types, functions, variables, etc.) inside it. Namespaces are used to organize code into groups in order to prevent name collisions.
  • Constructor - A special method of a class that initializes a newly created object. Whenever an object is created the constructor is called automatically.
  • Fixture - A known state that is used to run tests many times.
  • Type Hint - Type hinting is a solution to statically indicate the type of a value within your code. Providing hints to functions to only accept the given data type.
  • Type Cast - Changing an expression from one data type to another. For example, conerting an integer value into a floating point number.
  • Unit Test - Testing in which the smallest testable module of an application (functions, procedures, interfaces) are tested to ascertain if they are fit to use.
  • Static - Static methods and variables are not dependent on instantiation of an object. The static method or variable can be accessed without without instantiation of the class to which it belongs.
  • Class Method - A method which is bound to the class and not the object of the class. They have the access to the state of the class as it takes a class parameter that points to the class and not the object instance. It can modify a class state that would apply across all the instances of the class.
  • Facade - Design pattern. An object that serves as a front-facing interface masking more complex underlying or structural code.
  • Method - A procedure associated with a message and an object. A function within a class.
  • Property - A property is a class member that provides a flexible mechanism to read, write, or compute the value of a private field.
  • Abstraction - One of the four core concepts of OOP. Concept of hiding the implementation details of an object.
  • Encapsulation - One of the four core concepts of OOP. Bundling of data and methods into a single unit, most often a class.
  • Inheritance - One of the four core concepts of OOP. The mechanism of basing an object or class upon another object or class, retaining similar implementation. Deriving new classes from existing ones. A subclass is derived from a superclass (child and parent relationship).
  • Polymorphism - One of the four core concepts of OOP. Describes the concept that different classes can be used with the same interface. Allows us to perform a single action in different ways. Specifically, to allow an entinty (variable, function, or object) to have more than one form.
