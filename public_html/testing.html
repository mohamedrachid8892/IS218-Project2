AAA Testing Tutorial

This page provides the information needed to understand AAA testing in order to implement it into your programs.

What is Testing?

A test is a procedure that exercises a behavior to determine if the behavior functions correctly. There are several different kinds of tests, like unit tests, integration tests, or end-to-end tests, but all functional tests do the same basic thing: they try something and report PASS or FAIL.

Testing provides an empirical feedback loop for development. Thatâ€™s how testing keeps us safe. With tests, we know when things break. Without tests, coding can be dangerous. You don't want to deploy bugs.

How to Write Good Tests

You may be wondering "How do I write a good test?". There's a simple, but powerful, pattern used to write tests called AAA Testing. The three A's stand for Arrange-Act-Assert.

Arrange

Arrange inputs and targets. Arrange steps should set up the test case. Does the test require any objects or special settings? Does it need to prep a database? Does it need to log into a web app? Handle all of these operations at the start of the test.

Act

Act on the target behavior. Act steps should cover the main thing to be tested. This could be calling a function or method, calling a REST API, or interacting with a web page. Keep actions focused on the target behavior.

Assert

Assert expected outcomes. Act steps should elicit some sort of response. Assert steps verify the goodness or badness of that response. Sometimes, assertions are as simple as checking numeric or string values. Other times, they may require checking multiple facets of a system. Assertions will ultimately determine if the test passes or fails.

Example

Take a look at the following unit test written in Python. It tests Python's addition operator:

                
    def test_addition_operator():
        # Arrange
        value_a = 4
        value_b = 4

        # Act
        answer = value_a + value_b

        # Assert
        assert answer == 8

Although the example is extremely trivial, it should help you get a sense of how important tests are and how powerful they can be.

Using External Data in Testing

In Python, it is possible to run tests using data imported from an external file. The Python package Pandas allows you to work with other file types, such as Excel and CSV. It is incredibly powerful and flexible. In this tutorial, I will go over only how to read data from CSV files.

Your first step would be to install Pandas. Assuming you have Python installed already, open your terminal and enter the following command: pip install pandas. Pip is a package manager for Python. If you don't have Python installed, go to this link, download the file, and run it.

Once you are sure you have both Python and Pandas installed successfully, you can move on.

In order to start importing data into your Python program, do the following:

  1. Create your CSV file. This can be done in Excel. You can follow this template if you're unsure. Be sure to put the column headings in the first row. You can either input the values manually or generate random numbers. Be sure to save the file as a CSV and not an Excel file. Name the file data.csv.
  2. Enter the following code snippet at the top of your Python program: 
                            
    import pandas as pd
    You can now start using Pandas in your program.
  3. In order to read in the data, enter the following: 
                            
    df = pd.read_csv('.../data.csv')
    print(df)
    This code snippet reads in the data and prints it out. Be sure to include the absolute path to the CSV data file.

Now that we know how to import CSV files into our Python program, lets look at a sample program that utilizes this in unit testing.

Take a look at this program that reads in this csv file, called data.csv. The program ensures that the sum of all values equals result. 

                
    import pandas as pd

    data = pd.read_csv("data.csv")

    for a, b, c, d, e in  data.iterrows():
        assert (a + b + c + d) == e

This short program reads in the data.csv files, and iterates over each row in the file. For each row, it checks if the sum of the four values equals the result value. If it is false, the assertion will fail.

For more information on testing with CSV files, check out the Pandas documentation or this article and subarticles.

Running the Tests

Now that your tests are written, simply open the terminal and cd into the folder where your program is saved. Enter pytest into the terminal to have Pytest output whether your tests passed or not.
