Pylint

This page explains what Pylint is, and why it is important when developing Python programs.

What is Pylint?

Pylint is a powerful tool for all Python programmers. It is a tool that checks for errors in code, enforce the PEP 8 coding standard, look for type errors, recommend suggestions for how a code can be refactored, and offer details about a code's complexity. PEP 8 is a document that provides guidelines and best practices on how to write Python code. In short, Pylint is a fantastic tool that all Python programmers should be using to increase the quality of their code.

Using and getting started with Pylint is very easy. Read on to learn more.

Installing Pylint

Installing Pylint is simple enough. For Windows and OS X users simply enter pip install pylint into your terminal to install Pylint. For instructions on how to install Pylint for all users, go to this link.

Using Pylint

Now that Pylint is installed successfully on your computer, be sure that you have Pytest installed as well. If you don't check out the tutorial on Pytest.

Once you have Pytest and Pylint installed, it is simple enough to use. Simply open a terminal and navigate to the working directory of your Python program. Once there, enter pytest --pylint into your terminal. This will output any warnings or errors in your code/coding style.

Customizing Pylint

You can customize your Pylint, disabling error checking for specific errors, deactivate the evaluation score, change refactoring rules, etc. This is done by placing a .pylintrc file in your programs directory. A sample .pylintrc file can be found here.
